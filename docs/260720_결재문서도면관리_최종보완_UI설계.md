## 1. 개요 및 UI/UX 설계 방향
안녕하세요, **PROCPA**입니다.
이전 v4.1 설계서가 가졌던 아키텍처적 한계점, 즉 기획 문서 내에 Canvas, S3, HMAC 등 구현 기술 용어가 노출되었던 기술 결합 오염과 라우터 중심의 세분화된 경로 분류, 그리고 비인가 검증 화면(`/verify`)을 통한 외부 정보 누출 위협을 완벽히 인식하고 이를 대대적으로 교정했습니다.

구현 기술 용어를 철저하게 순수 기획 비즈니스 단어로 정제하고, 결재/탐색 기능 위주의 계층형 화면 묶음 구성, 1회용 토큰 연동 도면 정보 은닉 보안 검증, 그리고 오프라인 시간 경과별 다단계 신선도 격하 UX를 수립한 **결재 통합 문서도면관리 시스템 UI/UX 상세 설계서(v4.2)**를 제시합니다.

## 2. 기능 중심의 5대 화면 그룹 및 레이아웃 구성 (Feature-Driven UI Structure)
라우터 주소 위주의 세분화 구성을 폐지하고, 기획자와 QA 조직이 업무 흐름을 직관적으로 검증할 수 있도록 기능 단위 계층 구조로 화면을 재설계합니다.

### 2.1. [기능 그룹 A] 도면 및 문서 탐색 (DMS Explorer & PDF 상세 뷰어)

**1. 문서 탐색기 뷰 (DMS Explorer)**
  - **화면 구성:** 상단 다기능 검색 영역, 중앙 대용량 도면 리스트 그리드, 하단 페이징 영역으로 구성됩니다.
  - **다차원 고속 검색 및 개인화 검색 기능:**
    - **입력 필터 필드군:** 품번, 기종, Revision, 작성자, 부서, 보안등급, 결재상태, 라이프사이클상태, 등록일자(달력 범위), 태그.
    - **[검색 조건 저장(즐겨찾기)] 및 [공유 필터 등록]:** 현재 적용된 다차원 필터를 저장해 보관할 수 있으며, 타 부서원에게 공유하는 공유 필터 지정이 가능합니다.
    - **즐겨찾기 관리:** 필터 이름 변경, 필터 삭제, 그리고 정렬 기능이 탑재되어 개인화 검색을 보조합니다.
    - **[초기화] 및 [최근 검색어]:** 초기화 클릭 시 필터 조건을 즉시 원복하고, 검색창 포커스 시 최근 1시간 내 수행한 최근 검색 조건 5개를 바로가기 연동합니다.

**2. 도면 상세 뷰어 (Secure PDF Viewer)**
  - **화면 구성:** 좌측 3대 계보 정보 탭 패널과 우측 보안 도면 뷰포트로 분할된 스플릿 화면입니다.
  - **PDF 뷰어 7대 코어 피처 스펙:**
    - `Zoom` (배율 확대/축소), `Rotate` (도면 90도 회전), `Page Control` (페이지 넘김), `Thumbnail Side-panel` (좌측 축소판 리스트), `Find` (도면 텍스트 검색), `Secure Download` (권한 기반 난독화 파일 다운로드), `Secure Print` (보안 QR코드가 자동 합성된 인쇄 파일 출력).
  - **좌측 3대 탭:** [개정 히스토리] ➔ 계보 추적, [참고 문서 링크] ➔ 다대다 관계도 시각화, [결재 이력 원장] ➔ 결재 진행 변경 내역 목록 노출.

### 2.2. [기능 그룹 B] 결재 업무 포털 (Approval Gateway)

**1. 결재 통합 함 (To-Do / Outbox)**
  - **화면 구성:** [결재 대기 목록], [대결 수임 목록], [기안 진행 현황] 3대 영역을 탭으로 전환하여 관제하는 결재 포털.
  - **사용자 흐름:** `DRAFT` 또는 결재 반려(`REJECTED`), 취소(`CANCELLED`) 상태인 도면만 결재선 지정 및 드래그 앤 드롭 편집 기안 상신이 가능합니다.

**2. 결재 심사 및 결재선 편집 뷰**
  - **화면 구성:** 상신 정보 및 대결 사유 입력 폼, 순서 편집 패널.
  - **비즈니스 규칙:** 결재 진행 상태가 `UNDER_REVIEW`로 전이된 문서는 결재선 수정이 불가능하도록 인풋 폼이 비활성화 잠금 처리됩니다.

### 2.3. [기능 그룹 C] 모바일 간이 검증 (Token-Secured Mobile Verify)

**1. 1회용 보안 토큰 연동 검증 화면**
  - **화면 구성:** 비인가 외부인의 도면 정보 탈취(품번, 기종, 도면 이미지 노출에 의한 보안 사고)를 방지하기 위해 설계된 화면입니다.
  - **보안 정보 은닉 처리:** 실물 도면에 합성된 QR코드 스캔 시, 도면 ID가 노출되지 않는 **1회용 보안 난수 토큰** 경로로 인입되며, 화면 내에는 도면의 파일 링크, 품번, 규격 등 일체의 기밀 설계를 완전히 숨깁니다.
  - **판정 결과 배지 노출:** 오직 스캔한 실물 도면의 현재 유효성 상태 배지(`Green`/`Yellow`/`Red`) 및 후속 작업 지침("최신 도면으로 대체 인쇄 필요")만을 직관적으로 노출합니다.

**2. 오프라인 시간 경과별 다단계 신선도 배지 격하 모델**
  - 공장 지하 음영 지역 등으로 인한 단절 발생 시, 로컬 캐시 확인 경과 시간에 따라 신호등 인디케이터 배지를 미세하게 격하 조치합니다.
  - `Green (유효성 확인완료)`: 오프라인 경과 시간 0분 ~ 10분 이내 (실시간 신뢰성 보장).
  - `Yellow (유효성 확인 - 주의)`: 오프라인 경과 시간 10분 ~ 30분 이내 (약한 지연 경보).
  - `Orange (정보 지연 - 경고)`: 오프라인 경과 시간 30분 ~ 2시간 이내 (주의 경보, 확신도 격하).
  - `Red (검증 불가)`: 오프라인 경과 시간 2시간 이상 지속 시 (캐시 신뢰 기각, 작업 투입 금지 및 온라인 동기화 필수 안내).

### 2.4. [기능 그룹 D] 공정 단말 제어 (Terminal Supervision)
- **화면 구성:** 태블릿이 배치된 각 현장 단말기별 원격 새로고침 강제 명령 송출 패널 및 SSE 세션 연결 상태 관제 대시보드.

### 2.5. [기능 그룹 E] 시스템 운영 설정 (System Admin Gateway)
DMS의 신뢰성 높은 유지보수를 위해 단일 Admin을 4대 전문 관리 메뉴로 정립합니다.

**1. 사용자 및 역할/권한 관리 뷰 (User & Role Management Dashboard - `/admin/users`)**
  - **사용자 관리 메인 그리드:** 전체 사용자 목록, 소속 부서, 이메일, 계정 인증 출처(`LOCAL` / `SSO`), 보안 취급 등급(`GRADE_1`~`GRADE_3`), 부여된 역할 태그(`ROLE_USER`, `ROLE_APPROVER`, `ROLE_ADMIN`), 및 계정 상태(`ACTIVE`/`LOCKED`)를 한눈에 관제하는 테이블 UI.
  - **[신규 사용자 등록 모달]:** 아이디, 비밀번호(로컬 계정 시), 이메일, 이름, 소속 부서(드롭다운), 계정 유형(`LOCAL` / `SSO`), ABAC 보안등급, ABAC 담당기종(복수 선택)을 등록하는 입력 폼.
  - **[역할 매핑 및 권한 편집 팝업]:** 특정 사용자를 선택하여 `ROLE_USER` (일반 기안자), `ROLE_APPROVER` (결재 심사권자), `ROLE_ADMIN` (시스템 관리자) 역학 체크박스를 토글 매핑하고, ABAC 보안 속성(부서 변경, 보안등급 변경)을 즉시 수정 갱신하는 권한 제어 UI.

**2. Security & Access Control**
  - ABAC 주체 부서 코드 관리, 기종 코드 정의 및 전역 보안 규칙 위임 관리.

**3. System & Monitoring**
  - SSE 브라우저 멀티탭 커넥션 연결 현황 감시, 백엔드 데이터 파티셔닝 배치 스케줄러 모니터링, 시스템 오류 복구 통제.

**4. Storage & File Management**
  - 물리 보관소 사용 용량 관제 및 비동기 PDF 변환/OCR 문자 텍스트 추출 큐 처리 현황 대시보드.

## 3. 실물 종이 도면 인쇄 제어 및 보안 대응 UX

**1. 동기식 보안 QR코드 합성 인쇄**
  - 도면 뷰어 내에서 인쇄 버튼 클릭 시, 백엔드로부터 발급받은 **1회용 난수 토큰**과 현재 유효한 서명키 세대값(`kid`), 그리고 HMAC 암호화 서명이 탑재된 QR코드 이미지가 캔버스에 완전히 합성되어 렌더링이 종결될 때까지 화면에 [보안 인쇄 파일 준비 중...] 모달을 띄우고 브라우저 프린터 팝업을 일시 정지(Lock)시킵니다.

**2. 위변조 서명키 무효화(Revoked Key) 보안 예외 UI**
  - 모바일 간이 검증 스캔 시, 사용된 `kid` 서명키가 유출 사유로 인해 무효화 처리된 경우, 화면 전체에 적색 테두리 경보를 표출하고 **[보안 무효화 키 감지: 해당 도면은 인쇄 보안 키 유출로 무효화되었습니다. 즉시 폐기하고 최신 키로 재발급 인쇄하십시오. (REVOKED - Red)]**라는 굵은 예외 안내창을 강제로 노출하여 작업 오가공을 차단합니다.

## 4. 마치며 및 결론
본 UI/UX 상세 설계서(v4.2)에서는 기획서 내에 불필요하게 기재되어 있던 Canvas, S3, OCR Queue 등의 **구현 기술 단어를 보안 QR코드, 물리 파일 보관소, 텍스트 자동 인식 스케줄 등의 기획 단어로 완전 정제하여 아키텍처적 결합을 안전하게 해소**하였습니다.

동시에, 외부 유출을 원천 봉쇄하는 **1회용 토큰 연동 간이 검증 화면**과 **다단계 오프라인 캐시 신선도 격하 UX**, **7대 PDF 뷰어 피처**, 그리고 **3대 특화 어드민 메뉴**를 완비하여 기획 수준의 정합성을 최고 단계로 완결시켰습니다.

> [!check] v4.2 UI/UX 기획 정제 타결 요약
> - **기술 오염 제거:** Canvas ➔ 보안 합성 인쇄, S3 ➔ 물리 파일 보관소 등 기획 중심 단어로 전면 개칭
> - **경로 기획 그룹화:** 라우트 단위 표기를 배제하고 5대 비즈니스 기능 그룹 중심 계층 구성
> - **verify 보안 강화:** 1회용 난수 토큰 검증 경로 적용 및 도면 상세 정보 은닉 처리
> - **신선도 세분화:** Green(10분) ➔ Yellow(30분) ➔ Orange(2시간) ➔ Red(2시간 초과) 다단계 격하 탑재

이상으로 UI/UX 상세 설계서의 수정을 완결 지었습니다.
